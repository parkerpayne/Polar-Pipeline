Setup

This page will walk you though the entire process to set up a new worker.

1 - Dependencies

Install necessary dependencies using the following commands.

sudo apt install pigz
sudo apt install bedtools
sudo apt install tabix
sudo apt install celery
sudo apt install nfs-common
pip install tqdm
pip install psycopg2-binary
pip install bs4
sudo cpan DBI
sudo cpan Math::CDF
sudo cpan List::MoreUtils

2 - Minimap

Put the minimap2 folder in the home directory. You will now need to add the minimap folder to PATH. PATH is a linux environment variable. It is a string of paths separated by : that define what directories to search for executables in when a command is invoked. This variable is referenced by $PATH and can be defined in the .bashrc file in the home directory. Nano into the .bashrc and add the following at the end.

PATH="/new/minimap/path:$PATH"


This adds the minimap directory to the front of the PATH variable, meaning it will be searched first, improving speed of locating minimap when running the command. The following source command refreshes the environment, updating PATH.

source .bashrc

3 - Samtools

Place samtools path in the home directory like minimap. Add the samtools folder to PATH like so:

PATH="/new/samtools/path:$PATH"


Update the environment by running:

source .bashrc

4 - Nextflow

Install nextflow using the following commands:

curl -s https://get.nextflow.io | bash
sudo mv nextflow /usr/local/bin

5 - Vep

Install VEP and all plugins. The plugins are as follows:

  • LoFtool
  • Mastermind
  • CADD
  • Carol
  • Condel
  • pLI
  • PrimateAI
  • dbNSFP
  • REVEL
  • AlphaMissense
  • EVE
  • DisGeNET

There should be three folders in the home directory, ensembl-vep, .vep and vep-resources, the latter containing ALL extra files required by the plugins.

Inside both .vep and vep-resources, there will be a file at /home/{PCNAME}/.vep/Plugins/config/Condel/config/condel_SP.conf The first line of that file contains a path for some reason, and this path is necessary for sift and polyphen. I don't know why. You must replace the path with the path to the Condel folder. For instance, if the file's path is /home/i9a6000/.vep/Plugins/config/Condel/config/condel_SP.conf, you will define condel.dir as /home/i9a6000/.vep/Plugins/config/Condel. Do this for both locations.

6 - Shared Directories

Set up NFS for the resources. Create the NFS directory /mnt/pipeline_resources/. Connect it to the resources folder in the docker container located at ../polar-pipeline-docker/services/web/polarpipeline/resources.

7 - Set up final variables

Inside the resources folder, (/mnt/pipeline_resources on your worker) there is a file called variables.config.example. Open the file and configure the server's local ip, as well as the sudo password for each worker. The name should match the output of whoami .

Complete

The setup should at this point be complete, and the worker can be started by running the celery app from the /mnt/pipeline_resources directory:

celery -A tasks worker --concurrency=1 --prefetch-multiplier=1 --loglevel=INFO > $(whoami).log 2>&1
